# Шпаргалка: списки в Python

Краткий справочник по спискам.

---

## Создание

```python
db: list[ItemInDB] = []       # пустой список
db: list[ItemInDB] = [item1]  # список с одним элементом

db = []       # без аннотации - работает так же, но IDE не подсказывает
db = [item1]  # без аннотации - работает так же, но IDE не подсказывает
```

---

## Добавить элемент

```python
db.append(ItemInDB(id=1, name="Мышь", price=1200))
```

---

## Перебрать все элементы

```python
for item in db:
    print(item.id, item.name)
```

---

## Найти элемент по id

```python
for item in db:
    if item.id == item_id:
        return item
```

---

## Найти индекс и удалить

```python
for i, item in enumerate(db):  # i — индекс, item — сам объект
    if item.id == item_id:
        db.pop(i)               # удалить по индексу
        return
```

---

## Узнать максимальный id

```python
max((item.id for item in db), default=0)
# default=0 — если db пустой, вернёт 0, а не ошибку
```

---

## Длина списка

```python
len(db)  # количество элементов
```

---

## Получить элемент по индексу

```python
db[0]   # первый элемент
db[-1]  # последний элемент
```

---

## Индекс vs id - важно не путать

В этой задаче у каждого товара есть `id` - это поле объекта, которое мы сами задаём. Это **не** позиция в списке:

```python
db = [
    ItemInDB(id=1, name="Ноутбук", price=89900),  # индекс 0, id=1
    ItemInDB(id=2, name="Мышь",    price=1200),   # индекс 1, id=2
]

db[0]     # по индексу - всегда "Ноутбук"
db[0].id  # → 1

# После удаления первого элемента:
db.pop(0)
db[0]     # теперь "Мышь" - индекс сдвинулся, но id=2 остался
```

Поэтому в эндпоинтах ищем всегда по `item.id`, а не по позиции в списке.
